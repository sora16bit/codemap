# README screenshots

The READMEs reference two images that need to be captured manually
(headless browser screenshots can't be produced in the dev container —
the system is missing browser libs like `libnspr4`).

Capture both from a running dev server (`npm run dev`, then
<http://localhost:3000>) and save them here:

- **`hero-ky.png`** — analyze a larger, well-known repo so the map looks
  substantial. Suggested: `sindresorhus/ky` (52 files / 127 edges).
  Used as the hero image at the top of every README.
- **`self.png`** — analyze this repository itself (the "Dogfooding"
  section). Proves the tool runs on real code, including its own.

Keep them reasonably wide (e.g. 1400px) and crop to the graph area.
